# Checklist manuelle V0.2.1 — Canvas et première UX définitive

Ne cocher que les contrôles réellement réalisés sur un environnement humain.

## Nouvelle UX

- [ ] Toutes les bulles `?` sont visibles.
- [ ] Les structures sont nommées.
- [ ] La révélation fonctionne depuis la barre publique.
- [ ] Le nom apparaît après révélation.
- [ ] Les liens pertinents apparaissent progressivement.
- [ ] Les autres noms restent cachés.
- [ ] Le message « Ce style est déjà révélé. » fonctionne.
- [ ] L’erreur de recherche ne divulgue pas de liste.
- [ ] Le clavier mobile se ferme ou ne gêne pas le focus.
- [ ] Le menu inférieur est visible.
- [ ] Les toasts des futurs écrans s’affichent.

## Canvas

- [ ] Pas de pixellisation excessive.
- [ ] Netteté sur écran Retina.
- [ ] Redimensionnement.
- [ ] Portrait.
- [ ] Paysage.
- [ ] Pan.
- [ ] Zoom.
- [ ] Pinch.
- [ ] Focus.
- [ ] Sélection.

## Performance

- [ ] 20 nœuds.
- [ ] 50 nœuds.
- [ ] 100 nœuds.
- [ ] 200 nœuds.
- [ ] 300 nœuds.
- [ ] CPU au repos.
- [ ] Fluidité du pan.
- [ ] Fluidité du zoom.
- [ ] Fluidité de révélation.
- [ ] Changement de LOD.
- [ ] Onglet masqué.

## Comparaison

- [ ] Moteur Canvas.
- [ ] Moteur SVG historique.
- [ ] Différence de fluidité.
- [ ] Différence visuelle.
- [ ] Absence de régression logique.

## Appareils

- [ ] Chrome desktop.
- [ ] Firefox desktop si disponible.
- [ ] Safari desktop si disponible.
- [ ] iPhone Safari réel.
- [ ] Android réel ou émulation.
